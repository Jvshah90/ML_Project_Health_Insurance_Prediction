# ML_Project_Health_Insurance_Prediction

Live Deployment:
https://health-insurance-calculator.streamlit.app/

An interactive predictive system that estimates health insurance premiums based on user inputs. 
Designed to assist underwriting decisions through data-driven insights and deployed for real-time access. 

Project Topic : Predictive Health Insurance Premium Estimation System

Project Statement (Problem Statement)
The health insurance industry often relies on manual underwriting processes or basic rule-based systems to determine insurance premiums. These approaches can be time-consuming, inconsistent, and may not fully utilize available data for accurate risk assessment. There is a need for a data-driven system that can intelligently predict health insurance premiums based on key individual attributes such as age, BMI, smoking habits, and medical history. The goal is to improve accuracy, reduce manual effort, and enable faster decision-making for underwriters.

Problem Identification
From an analysis perspective, the key problems observed are:

1. Manual & Time-Consuming Underwriting
Traditional methods require human evaluation of each case.
Leads to delays in generating insurance quotes.

2. Inconsistent Pricing Decisions
Different underwriters may interpret risk differently.
Results in lack of standardization.

3. Limited Use of Data
Existing systems may not fully leverage historical data patterns.
Missed opportunity for predictive insights.

4. Accuracy Challenges
Premium estimation may not always reflect actual risk.
Can lead to financial loss or customer dissatisfaction.

5. Lack of Accessibility
Tools are not always available remotely or in real-time.
Underwriters need a centralized, easy-to-use system.
Proposed Solution (R&D Perspective)

Develop a machine learning-based predictive system that:

Uses historical data to learn patterns in premium calculation
Predicts insurance premiums with high accuracy
Provides a user-friendly interface for real-time predictions
Can be deployed on cloud infrastructure for remote accessibility

Objectives
Primary Objectives:
Build a predictive model with high accuracy (target >97%)
Ensure prediction error stays within acceptable limits (±10% for most cases)

Secondary Objectives:
Automate the underwriting support process
Improve consistency in premium calculation
Enable faster insurance quote generation
Provide a scalable and secure deployment

Scope of Work (Simplified & Analytical)
1. Data Understanding & Preparation
Collect structured health insurance data
Clean missing or inconsistent values
Perform exploratory analysis to identify patterns

2. Model Development
Train multiple machine learning models
Compare performance using evaluation metrics
Optimize the best-performing model

3. Deployment
Host the model on a cloud platform
Ensure scalability and security

4. Application Layer
Build an interactive interface (e.g., Streamlit)
Allow users to input data and get predictions instantly

5. Testing & Validation
Validate model performance using real-world scenarios
Ensure reliability and robustness

6. Documentation & Knowledge Transfer
Provide clear documentation
Train end-users on system usage

Expected Outcomes
A reliable predictive model for insurance premiums
A deployed application accessible from anywhere
Reduced dependency on manual underwriting
Faster and more consistent decision-making

Future Scope (Phase 2 Idea)
Automation of complete insurance quote processing (Straight-Through Processing - STP)
